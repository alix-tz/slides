Encodage numérique¶

L'encodage dsigne la transformation de donnes d'un format un autre.

Le degr zro de l'encodage numrique, c'est la transformation de donnes en valeurs binaires qui peuvent tre traites l'chelle lectronique (teint/allum). On reprsente ces tats par un 1 ou un 0.

  • un bit est une instance de 0 ou de 1
  • un train de bits (ou byte) est un groupe compos d'une suite de bits
  • un octet est un groupe de 8 bits

Un fichier reprsent par 100 octets est donc plus riche, plus complexe qu'un fichier reprsent par 10 octets.

On calcule la taille d'un fichier numrique partir du nombre d'instances de 0 et de 1 ncessaires sa reprsentation numrique.

Tout document conservé sur un ordinateur est constitué d'une série de 0 et de 1¶

Mémoire¶

La taille d'un disque dur dfinit le nombre de bits disponibles pour encoder des informations simultanment.

Par exemple, un disque dur de 50 Go de mmoire :

  • est gal 50 000 000 000 octets (cinquante milliards)
  • est gal 400 000 000 000 bits (quatre cents milliards)

La taille d'un fichier dfinit le nombre de bits ncessaire son encodage.

Par exemple, un fichier d'une taille de 30 Ko :

  • ncessite 30 x 1 000 x 8 bits
  • contient donc un total de 160 000 instances de 1 ou 0

Normes d'encodage¶

Les normes d'encodages dfinissent les clefs d'encodage et de dcodage des donnes lorsque l'on passe d'une forme intelligible pour l'humain une forme lisible par la machine.

En fonction du type de fichier encod (texte, image, vido, etc.), il existe diffrents paysages de normes et de modes de reprsentation.

Encodage des données textuelles¶

Un document numrique textuel contient du texte, c'est--dire une suite de caractres alphanumriques et de symboles.

La norme d'encodage d'un texte == une table de correspondance entre un groupe de bits et un caractre alphanumrique ou un symbole.

La norme d'encodage dfinit donc la liste des caractres disponibles (character set).

En fonction de la norme, la reprsentation d'un caractre peut s'tendre sur un ou plusieurs octets.

Les normes les plus connues pour des textes en franais sont :

  • ASCII (1963)
  • ISO 8859-1 (1986)
  • UTF-8 (1996)

American Standard Code for Information Interchange

Chaque caractre est reprsent par 7 bits empaquets dans 1 octet

Le jeu de caractres est compos de 95 lments (pas de caractres accentus) :

Universal Character Set Transformation Format + base d'encodage (8, 16, 32, ...)

En fonction de la base d'encodage, un caractre peut tre reprsent par 1, 2, 3 ou 4 octets.

UTF est rtrocompatible avec ASCII, mais pas avec ISO 8859-1. Il s'inscrit dans le cadre tabli par Unicode qui vise faciliter l'change de textes numriques en assurant leur compatibilit.

Unicode dfinit plus de 137 000 caractres, issus de multiples alphabets, et prvoit aussi des symboles comme les emojis.

Encodage des images numériques¶

Il existe deux grandes catgories d'images :

  • les images matricielles;

  • les images vectorielles.

Images matricielles¶

Les images matricielles (ou bitmap) sont des tableaux de points 2 dimensions (hauteur, largeur)

La dfinition d'une image indique les dimensions de sa matrice. Une image de dimensions "32 x 21" est donc un tableau de 32 colonnes (largeur) et 21 lignes (hauteur). L'image est reprsente par 672 cases (ou pixel).

Les pixels ont des proprits (coordonnes, couleur).

La rsolution d'une image indique simplement la taille de l'image lorsqu'elle est reprsente dans des dimensions matrielles. L'unit de mesure de la rsolution dpend de l'unit de mesure matrielle : DPI (dot per inch) -> indique le nombre de pixels utilis pour reprsenter 1 pouce (2,54 cm).

La taille d'un fichier image ne dpend pas uniquement du nombre de pixels qui la compose mais aussi de la quantit d'informations contenues dans un pixel.

Images matricielles et encodage de la couleur¶

L'encodage de la couleur dans les systmes informatiques pourrait faire l'objet d'un cours part entire.

Les principaux modes de reprsentation d'une image colore (ou non) sont :

  • le mode binaire (noir ou blanc),
  • le niveau de gris (intensit du noir entre 0 et 255*),
  • le mode RGB (synthse de trois couches monochromes contenant des valeurs d'intensit situes entre 0 et 255*).

Une image en mode RGB est donc compose de l'assemblage de 3 matrices.

* 0-255 quand la valeur est enregistre sur 1 octet.

Un pixel contient la valeur d'intensit des points situs aux mmes coordonnes sur chaque couche de couleur.

Le poids des images¶

Le poids d'une image matricielle dpend en partie de la dfinition de l'image (nombre de pixels) et du mode de gestion des couleurs (nombre de couches de couleur)

Le poids d'une image matricielle dpend aussi du format utilis pour sa sauvegarde.

Images vectorielles¶

Une image vectorielle n'est pas constitue de pixels, mais de texte

Faite d'objets simples (segments, polygones, courbes de Bziers) dfinis par des proprits (coordonnes, couleur de contour ou de remplissage, angle de rotation)

On peut modifier individuellement chacun de ces objets sans affecter l'ensemble

On peut animer une image vectorielle car les attributs d'un objet vectoriel peuvent tre affects par une variable de temps

Une image vectorielle n'a pas de rsolution : elle est dfinie uniquement si l'on transforme l'image vectorielle en image matricielle

Le format d'images vectorielles le plus commun est SVG (Scalable Vector Graphics) : soit du XML

On peut ouvrir une image vectorielle avec un diteur d'image (comme Illustrator ou Inkscape) ou avec un diteur de texte.

Une image dans un fichier texte¶

On peut ouvrir une image vectorielle avec un diteur de texte ou un diteur d'image, on ne peut ouvrir une image matricielle qu'avec un diteur d'image
